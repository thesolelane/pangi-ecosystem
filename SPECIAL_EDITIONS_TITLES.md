# PANGI Special Editions - Titled 1-of-1 NFTs

## Overview

Special Edition PANGI NFTs are ultra-rare 1-of-1 pieces with unique titles and artwork. Unlike the main collection's matching pairs, these are standalone NFTs that can be either pangopups OR adults (not both).

---

## Key Characteristics

- **Supply**: ~20-50 total (flexible based on titles)
- **Rarity**: 1-of-1 unique NFTs
- **Stage**: Either Pangopup OR Adult (no matching pairs)
- **Artwork**: Custom, not from main collection trait system
- **Titles**: Royal Court, Government, Legendary roles
- **Distribution**: Promotional, events, top stakers, auctions

## Artwork Guidelines

### Pangopup Titles
- Soft, pale scales (biological accuracy)
- Clinging to mother or playful pose
- Title-appropriate accessories (small crown, cape, etc.)
- Innocent, curious expression
- Lighter, softer color palette

### Adult Titles
- Hardened, dark scales
- Confident, authoritative stance
- Full title regalia (crown, armor, robes, etc.)
- Battle-ready or regal pose
- Rich, bold color palette

### Common Elements
- Obsidian Claw clan insignia
- Title-specific background
- Unique aura/glow effect
- Custom trait combinations
- 1-of-1 artwork (no trait system)

## Technical Implementation

### Smart Contract

```rust
#[account]
pub struct SpecialEditionNFT {
    pub nft_mint: Pubkey,
    pub title: String,              // "King", "Queen", etc.
    pub category: TitleCategory,    // Royal, Government, Legendary
    pub stage: Stage,               // Pangopup or Adult
    pub governance_weight: u32,     // 100-1000
    pub staking_multiplier: u16,    // 150-500 (basis points)
    pub unique_id: String,          // "KING-001"
    pub is_1_of_1: bool,           // Always true
    pub matching_pair: Option<Pubkey>, // Always None
}

#[derive(AnchorSerialize, AnchorDeserialize, Clone)]
pub enum TitleCategory {
    RoyalCourt,
    Government,
    Legendary,
}
```

### Minting Process

1. Admin-only minting (no public mint)
2. Custom metadata per title
3. Unique artwork upload
4. Governance weight assignment
5. Distribution tracking
